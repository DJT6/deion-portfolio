Technical Deep Dive

Architecture, workflows, and AWS service integration for the Healthcare Appointment Booking System.

Architecture

The system follows a three-tier cloud architecture on AWS:

  • Patients access the app through a frontend hosted on S3 and delivered globally with CloudFront.
  • A Node.js/Express backend runs on a t2.micro EC2 instance inside a VPC, handling appointment booking and management.
  • Data is stored in DynamoDB Free Tier (or SQLite on EC2 for testing), simulating the persistence layer.
  • IAM roles and security groups enforce least-privilege access and protect resources.
  • CloudWatch collects logs and metrics, providing monitoring and visibility into the system.

S3 → CloudFront → EC2 (API) → DynamoDB → CloudWatch.
