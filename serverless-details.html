Technical Deep Dive

Architecture, workflows, and AWS service integration for the Serverless Contact Form.

Architecture

The application follows an event-driven, serverless design that connects multiple AWS services:

  • User submits the contact form → triggers API Gateway.
  • Lambda function validates the data and:
    • Stores the submission in DynamoDB.
    • Saves a JSON copy in S3 for analytics.
    • Publishes an alert to SNS.
    • Emits custom metrics to CloudWatch.
  • SNS notifies the owner when new submissions are received.
  • CloudWatch captures logs, tracks metrics, and monitors system health.

API Gateway → Lambda → DynamoDB/S3 → SNS → CloudWatch.
