Technical Deep Dive

Architecture, workflows, and AWS service integration for the Serverless Contact Form.

Architecture

The system uses an **event-driven serverless architecture**:

  • User submits the contact form → triggers **API Gateway**.
  • **Lambda function** processes the input and stores it in **DynamoDB**.
  • **SNS** sends an email notification to the owner.
  • **CloudWatch** monitors metrics, logs, and triggers alarms for errors.

Optional: Include a diagram showing API Gateway → Lambda → DynamoDB → SNS → CloudWatch.

Features & Highlights

  • Secure **IAM least-privilege roles** for Lambda, DynamoDB, and SNS.
  • **CloudWatch dashboards** and alarms for observability.
  • **Event-driven notifications** using SNS email.
  • Frontend is responsive with a **thank-you confirmation page**.
  • Deployed using **Infrastructure-as-Code** (CloudFormation/Terraform).
